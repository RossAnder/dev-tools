# review-plan trigger utterances

Utterances below are used to evaluate whether the `review-plan` skill description activates on the right inputs. Positive examples should trigger it; negative examples should not.

## Positive

- "review the plan at docs/plans/foo.md"
- "audit the plan for risk and gaps"
- "check docs/plans/bar.md for missing tasks"
- "is the prod-prep plan feasible?"
- "critique docs/plans/queue-refactor/ for executability"
- "run a feasibility review on the plan at docs/plans/storage-migration.md"
- "can you audit docs/plans/hejlsberg/ for stale references and missing scope?"

## Negative

- "review this PR"
- "what do you think of my approach?"
- "can you read this plan aloud?"
- "does this function look right?"
- "show me the plan"
- "create a new plan for the queue refactor"
- "update the plan at docs/plans/foo.md with the latest progress"
